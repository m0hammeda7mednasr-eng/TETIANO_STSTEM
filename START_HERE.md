# 🚀 ابدأ من هنا - دليل البداية السريع

## ⚡ الخطوات الأساسية (5 دقائق)

### 1️⃣ تثبيت وتشغيل المشروع

```bash
# تثبيت جميع المكتبات
npm run install-all

# تشغيل Frontend و Backend معاً
npm run dev
```

سيعمل المشروع على:

- Frontend: http://localhost:3000
- Backend: http://localhost:5000

---

### 2️⃣ إعداد قاعدة البيانات

افتح Supabase SQL Editor ونفذ الملف:

```sql
-- انسخ والصق محتوى: SETUP_SAFE.sql
-- هذا الملف آمن ولا يحذف البيانات الموجودة
```

**ملاحظة:** إذا واجهت خطأ في جدول permissions، نفذ:

```sql
-- انسخ والصق محتوى: FIX_PERMISSIONS_TABLE.sql
```

---

### 3️⃣ معرفة حساب المدير

افتح Supabase SQL Editor ونفذ:

```sql
SELECT id, email, name, role FROM users WHERE role = 'admin';
```

**إذا لم يظهر أي نتيجة:**

1. سجل حساب جديد من http://localhost:3000/register
2. نفذ هذا الأمر (استبدل البريد):

```sql
UPDATE users
SET role = 'admin'
WHERE email = 'your-email@example.com';
```

## 🎯 ما يجب أن تراه

### المدير (Admin) يرى:

✅ **في Sidebar:**

- لوحة التحكم
- الطلبات
- المنتجات
- العملاء
- إدارة المستخدمين ← **الصفحة الرئيسية للمدير**
- الإعدادات
- زر تسجيل الخروج

❌ **لا يرى:**

- تقاريري اليومية (لأن المدير لا يكتب تقارير)

✅ **في Dashboard:**

- الإحصائيات (مبيعات، طلبات، منتجات، عملاء)
- طلبات الصلاحيات المعلقة
- التقارير اليومية الأخيرة
- رسوم بيانية

✅ **في صفحة إدارة المستخدمين (/users):**

- تبويب المستخدمين: إضافة، تعديل، حذف
- تبويب طلبات الصلاحيات: موافقة، رفض
- تبويب التقارير اليومية: مشاهدة تقارير الموظفين

---

### المستخدم العادي (User) يرى:

✅ **في Sidebar:**

- الصفحات التي لديه صلاحية عليها فقط
- تقاريري اليومية ← **يكتب تقاريره اليومية**
- زر "طلب صلاحية" (أصفر في الأسفل)

✅ **في صفحة تقاريري اليومية (/my-reports):**

- كتابة تقرير جديد
- إضافة عنوان، وصف، مهام منجزة
- رفع ملفات (صور، Excel، PDF، Word)
- حفظ كمسودة أو إرسال

✅ **في صفحة طلب صلاحية (/request-access):**

- اختيار الصلاحية المطلوبة
- كتابة سبب الطلب
- متابعة حالة الطلب

---

## 🔧 استكشاف الأخطاء السريع

### المشكلة: لا أرى صفحة معينة

**الحل:**

1. تحقق من دورك (admin أو user)
2. امسح الـ cache: `Ctrl + Shift + R`
3. تحقق من الصلاحيات في قاعدة البيانات

### المشكلة: خطأ في تسجيل الدخول

**الحل:**

1. تحقق من أن Backend يعمل على port 5000
2. تحقق من ملف `backend/.env` يحتوي على بيانات Supabase الصحيحة

### المشكلة: لا تظهر البيانات

**الحل:**

1. تحقق من تنفيذ جميع ملفات SQL
2. تحقق من الـ console في المتصفح (F12)
