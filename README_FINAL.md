## 🚀 البداية السريعة (3 خطوات فقط!)

### 1. التثبيت والتشغيل

```bash
npm run install-all
npm run dev
```

### 2. إعداد قاعدة البيانات

افتح Supabase SQL Editor وانسخ والصق محتوى:

```
SETUP_SAFE.sql
```

**إذا واجهت خطأ في جدول permissions:**

```
FIX_PERMISSIONS_TABLE.sql
```

### 3. معرفة حساب المدير

في Supabase SQL Editor:

```sql
SELECT id, email, name, role FROM users WHERE role = 'admin';
```

**إذا لم يظهر نتيجة:**

```sql
UPDATE users SET role = 'admin' WHERE email = 'your-email@example.com';
```

**في Sidebar:**

- ✅ لوحة التحكم

- ✅ المنتجات
- ✅ العملاء
- ✅ **إدارة المستخدمين** ← الصفحة الرئيسية

**في Dashboard:**

- الإحصائيات
- **طلبات الصلاحيات المعلقة** 🆕
- **التقارير اليومية الأخيرة** 🆕
- رسوم بيانية

**في صفحة /users:**

- تبويب المستخدمين: إضافة، تعديل، حذف
- تبويب طلبات الصلاحيات: موافقة، رفض
- تبويب التقارير اليومية: مشاهدة تقارير الموظفين

### 👨‍💻 المستخدم العادي (User)

**في Sidebar:**

- الصفحات التي لديه صلاحية عليها فقط
- ✅ **تقاريري اليومية** (دائماً)
- ✅ زر **"طلب صلاحية"** (أصفر في الأسفل)

**في صفحة /my-reports:**

- كتابة تقرير يومي جديد
- إضافة عنوان، وصف، مهام منجزة
- رفع ملفات (صور، Excel، PDF، Word)
- حفظ كمسودة أو إرسال

**في صفحة /request-access:**

- اختيار الصلاحية المطلوبة
- كتابة سبب الطلب
- متابعة حالة الطلب

## 🔄 سير العمل

### إضافة موظف جديد

```
المدير → /users → "إضافة مستخدم جديد" → ملء البيانات → اختيار الصلاحيات → حفظ
```

### طلب صلاحية جديدة

```
الموظف → /request-access → اختيار الصلاحية → كتابة السبب → إرسال
↓
المدير يرى إشعار في Dashboard
↓
المدير → /users → "طلبات الصلاحيات" → موافقة/رفض
↓
الموظف يرى النتيجة + الصفحة تظهر في Sidebar
```

### كتابة تقرير يومي

```
الموظف → /my-reports → "إضافة تقرير" → ملء التفاصيل → رفع ملفات → إرسال
↓
المدير يرى التقرير في Dashboard
↓
المدير → /users → "التقارير اليومية" → عرض التفاصيل
```

## 🛠️ استكشاف الأخطاء

### لا أرى صفحة معينة

```
1. تحقق من دورك (admin أو user)
2. امسح الـ cache: Ctrl + Shift + R
3. تحقق من الصلاحيات في قاعدة البيانات
```

### خطأ في تسجيل الدخول

```
1. تحقق من Backend يعمل على port 5000
2. تحقق من backend/.env يحتوي على بيانات Supabase
```

### لا تظهر البيانات

```
1. تحقق من تنفيذ COMPLETE_SETUP.sql
2. افتح Console المتصفح (F12) وشاهد الأخطاء
```
