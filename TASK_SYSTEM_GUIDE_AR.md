# دليل نظام المهام والنشاطات وصافي الربح

## التحديثات الجديدة

تم إضافة 3 أنظمة جديدة للمشروع:

### 1. نظام المهام (Tasks System)

#### للأدمن (Admin):

- **صفحة المهام** (`/tasks`): إدارة كاملة للمهام
  - إضافة مهام جديدة وتعيينها للمستخدمين
  - تحديد الأولوية (منخفضة، متوسطة، عالية، عاجلة)
  - تحديد تاريخ الاستحقاق
  - تعديل وحذف المهام
  - عرض المهام في 3 أعمدة: قيد الانتظار، قيد التنفيذ، مكتملة

#### للمستخدم العادي (User):

- **صفحة مهامي** (`/my-tasks`): عرض المهام المعينة له فقط
  - عرض المهام المعينة من المدير
  - تحديث حالة المهمة (قيد الانتظار، قيد التنفيذ، مكتملة)
  - عرض تفاصيل المهمة والموعد النهائي
  - لا يمكن حذف أو تعديل المهام

### 2. سجل النشاطات (Activity Log)

#### للأدمن فقط:

- **صفحة سجل النشاطات** (`/activity-log`): متابعة جميع الأنشطة في النظام
  - عرض جميع العمليات (إنشاء، تحديث، حذف)
  - تصفية حسب نوع العنصر (منتجات، طلبات، عملاء، مستخدمين، مهام)
  - إحصائيات شاملة:
    - إجمالي النشاطات
    - عمليات الإنشاء
    - عمليات التحديث
    - عمليات الحذف
  - عرض تفاصيل كل نشاط (المستخدم، التاريخ، الوقت، النوع)

#### التسجيل التلقائي:

- يتم تسجيل جميع العمليات تلقائياً عبر Database Triggers
- يتم تسجيل: المنتجات، الطلبات، المهام

### 3. صافي الربح (Net Profit)

#### للأدمن فقط:

- **صفحة صافي الربح** (`/net-profit`): إدارة أسعار التكلفة وحساب الأرباح
  - إدخال سعر التكلفة لكل منتج
  - حساب الربح تلقائياً: (سعر البيع - سعر التكلفة)
  - حساب هامش الربح: ((سعر البيع - سعر التكلفة) / سعر التكلفة × 100)
  - إحصائيات شاملة:
    - إجمالي الإيرادات
    - إجمالي التكاليف
    - صافي الربح
    - متوسط هامش الربح
  - بحث عن المنتجات
  - تعديل سريع لأسعار التكلفة

## قاعدة البيانات

### جداول جديدة:

#### 1. tasks

- id (UUID)
- title (VARCHAR)
- description (TEXT)
- assigned_to (UUID) - المستخدم المعين له
- assigned_by (UUID) - المدير الذي عين المهمة
- priority (VARCHAR) - low, medium, high, urgent
- status (VARCHAR) - pending, in_progress, completed, cancelled
- due_date (DATE)
- completed_at (TIMESTAMP)
- created_at (TIMESTAMP)
- updated_at (TIMESTAMP)

#### 2. activity_log

- id (UUID)
- user_id (UUID)
- action (VARCHAR) - created, updated, deleted, viewed
- entity_type (VARCHAR) - product, order, customer, user, task
- entity_id (UUID)
- entity_name (VARCHAR)
- details (JSONB)
- ip_address (VARCHAR)
- user_agent (TEXT)
- created_at (TIMESTAMP)

#### 3. task_comments

- id (UUID)
- task_id (UUID)
- user_id (UUID)
- comment (TEXT)
- created_at (TIMESTAMP)

### تحديث جدول products:

## API Endpoints الجديدة

### Tasks:

- `GET /api/tasks` - جلب المهام (Admin: كل المهام، User: مهامه فقط)
- `GET /api/tasks/:id` - جلب مهمة واحدة
- `POST /api/tasks` - إنشاء مهمة جديدة (Admin فقط)
- `PUT /api/tasks/:id` - تحديث مهمة
- `DELETE /api/tasks/:id` - حذف مهمة (Admin فقط)
- `GET /api/tasks/:id/comments` - جلب تعليقات المهمة
- `POST /api/tasks/:id/comments` - إضافة تعليق

### Activity Log:

- `GET /api/activity-log` - جلب سجل النشاطات (Admin فقط)
- `GET /api/activity-log/entity/:type/:id` - جلب نشاطات عنصر معين
- `GET /api/activity-log/stats` - إحصائيات النشاطات (Admin فقط)
- `POST /api/activity-log` - تسجيل نشاط يدوي

### Products:

- `PUT /api/dashboard/products/:id` - تحديث منتج (لإضافة cost_price)

## ملاحظات مهمة

1. **الصلاحيات**:

   - تم إضافة Indexes على الجداول لتحسين الأداء
   - يتم جلب آخر 100 نشاط فقط بشكل افتراضي
