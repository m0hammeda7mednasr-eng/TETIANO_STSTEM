# 🔧 حل مشاكل حساب المدير

## ❌ المشاكل الشائعة

### المشكلة 1: المدير يرى "طلب صلاحية"

**السبب:** الـ cache القديم في المتصفح

**الحل:**

```
1. اضغط Ctrl + Shift + R (لمسح الـ cache)
2. أو اعمل Logout ثم Login تاني
3. أو افتح المتصفح في وضع Incognito
```

---

### المشكلة 2: المدير يرى "تقاريري اليومية"

**السبب:** الـ cache القديم أو الدور مش متحدث

**الحل:**

```
1. تأكد إن دورك admin في قاعدة البيانات:
   SELECT email, role FROM users WHERE email = 'your-email@example.com';

2. إذا كان user، حوله لـ admin:
   UPDATE users SET role = 'admin' WHERE email = 'your-email@example.com';

3. اعمل Logout ثم Login تاني
4. اضغط Ctrl + Shift + R
```

---

### المشكلة 3: Dashboard مش بيعرض الطلبات المعلقة والتقارير

**السبب:** مفيش بيانات أو Backend مش شغال

**الحل:**

```
1. تأكد إن Backend شغال على port 5000
2. افتح Console المتصفح (F12) وشوف الأخطاء
3. تأكد إن فيه موظفين كتبوا تقارير أو طلبوا صلاحيات
4. جرب تعمل Refresh للصفحة
```

## 🛠️ خطوات الإصلاح الكاملة

### الخطوة 1: تأكد من الدور في قاعدة البيانات

```sql
-- شوف دورك
SELECT email, role FROM users WHERE email = 'your-email@example.com';

-- إذا كان user، حوله لـ admin
UPDATE users SET role = 'admin' WHERE email = 'your-email@example.com';
```

### الخطوة 2: امسح الـ cache

```
1. اضغط Ctrl + Shift + Delete
2. اختر "Cached images and files"
3. اضغط Clear data
```

### الخطوة 3: اعمل Logout ثم Login

```
1. اضغط زر "تسجيل الخروج"
2. سجل دخول تاني
3. تأكد إن Sidebar صح
```

### الخطوة 4: تحقق من Backend

```bash
# تأكد إن Backend شغال
cd backend
npm start

# يجب أن يظهر:
# Server running on port 5000
```

### الخطوة 5: تحقق من Console

```
1. اضغط F12
2. اذهب لـ Console
3. شوف لو فيه أخطاء حمراء
4. إذا فيه خطأ في /api/access-requests أو /api/daily-reports
   معناها Backend مش شغال صح
```

## 🎯 السيناريو الكامل للمدير

### 1. تسجيل الدخول

```
1. افتح http://localhost:3000/login
2. سجل دخول بحساب المدير
3. يجب أن تنتقل لـ /dashboard تلقائياً
```

### 2. في Dashboard

```
1. شاهد الإحصائيات
2. شاهد الطلبات المعلقة (إذا كان فيه)
3. شاهد التقارير الأخيرة (إذا كان فيه)
4. اضغط "عرض جميع الطلبات" للانتقال لـ /users
```

### 3. في صفحة /users

```
1. تبويب المستخدمين:
   - اضغط "إضافة مستخدم جديد"
   - املأ البيانات
   - اختر الصلاحيات
   - احفظ

2. تبويب طلبات الصلاحيات:
   - شاهد الطلبات المعلقة
   - اضغط "موافقة" أو "رفض"
   - أضف ملاحظات

3. تبويب التقارير اليومية:
   - شاهد تقارير الموظفين
   - اضغط "عرض التفاصيل"
   - شاهد المرفقات
```

## 🚨 إذا لم تنجح كل الحلول

### الحل النهائي:

```bash

# 2. امسح node_modules
rm -rf node_modules

# 3. أعد التثبيت
npm run install-all

# 4. شغل من جديد
npm run dev

# 5. في قاعدة البيانات، تأكد من الدور:
UPDATE users SET role = 'admin' WHERE email = 'your-email@example.com';

# 6. افتح المتصفح في وضع Incognito
# 7. سجل دخول
```

- `POST /api/users/create` - إضافة مستخدم جديد
