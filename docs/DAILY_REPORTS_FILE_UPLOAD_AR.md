# ✅ إصلاح التقارير اليومية ورفع الملفات

## المشاكل اللي اتحلت

### 1️⃣ التقارير اليومية - رفع الملفات

**المشكلة:**

- الموظفين مش قادرين يرفعوا ملفات (PDF, Excel, صور) مع التقارير
- الملفات كانت بتتحفظ في الـ state بس، مش على السيرفر

**الحل:**

- إضافة `multer` لرفع الملفات
- إضافة `FileUploadService` للتعامل مع Supabase Storage
- تعديل الـ backend routes لدعم رفع الملفات
- تعديل الـ frontend لإرسال الملفات بـ FormData

### 2️⃣ المهام - خطأ "غير مصرح"

**المشكلة:**

- الأدمن مش قادر يضيف مهام جديدة
- بيظهر خطأ "غير مصرح" رغم إنه أدمن

**الحل:**

- إصلاح التحقق من الـ role في backend
- إضافة error handling أفضل
- إضافة console logs للـ debugging

---

## الملفات المعدلة

### Backend:

#### 1. `backend/src/services/fileUploadService.js` (جديد!)

- خدمة رفع الملفات إلى Supabase Storage
- دعم رفع ملف واحد أو عدة ملفات
- دعم حذف الملفات
- إنشاء bucket تلقائياً

#### 2. `backend/src/routes/dailyReports.js`

- إضافة multer middleware
- تعديل POST endpoint لدعم رفع الملفات
- تعديل PUT endpoint لدعم رفع ملفات إضافية
- دعم أنواع الملفات: صور، PDF، Excel، Word
- حد أقصى لحجم الملف: 10MB

#### 3. `backend/src/routes/tasks.js`

- إصلاح التحقق من role الأدمن
- إضافة error handling أفضل
- إضافة console logs للـ debugging

### Frontend:

#### 4. `frontend/src/pages/MyReports.jsx`

- تعديل handleSubmit لاستخدام FormData
- تعديل handleFileUpload لحفظ الملف الفعلي
- إرسال الملفات مع Content-Type: multipart/form-data

#### 5. `frontend/src/pages/Users.jsx`

- إضافة عرض الملفات المرفقة في تاب التقارير اليومية
- روابط لتحميل الملفات
- أيقونات للملفات

## خطوات التثبيت

### 1. تثبيت multer

```bash
cd backend
npm install multer
```

### 2. إعادة تشغيل Backend

```bash
npm start
```

### 3. إعداد Supabase Storage (اختياري)

إذا كنت تريد استخدام Supabase Storage:

1. اذهب إلى Supabase Dashboard
2. اذهب إلى Storage
3. سيتم إنشاء bucket تلقائياً باسم `daily-reports-attachments`

**ملاحظة:** إذا لم يكن Supabase Storage متاحاً، يمكنك استخدام حل بديل:

- حفظ الملفات محلياً في مجلد `uploads/`
- استخدام خدمة سحابية أخرى (AWS S3, Cloudinary, إلخ)

## كيفية الاستخدام

### للموظفين:

#### 1. كتابة تقرير يومي مع ملفات:

1. اذهب إلى **"تقاريري اليومية"**
2. اضغط **"تقرير جديد"**
3. املأ البيانات:
   - التاريخ
   - عنوان التقرير
   - الوصف
   - المهام المنجزة
   - ملاحظات إضافية
4. **ارفع الملفات:**
   - اضغط على "اختر ملفات"
   - اختر الملفات (صور، PDF، Excel، Word)
   - يمكنك رفع حتى 10 ملفات
   - الحد الأقصى لكل ملف: 10MB
5. اضغط **"حفظ التقرير"**

#### 2. أنواع الملفات المدعومة:

- **صور:** JPG, PNG, GIF, WebP
- **PDF:** ملفات PDF
- **Excel:** XLS, XLSX
- **Word:** DOC, DOCX

#### 3. عرض الملفات المرفقة:

- في صفحة "تقاريري اليومية"
- كل تقرير يعرض عدد الملفات المرفقة
- يمكنك تحميل الملفات بالضغط عليها

---

### للأدمن:

#### 1. عرض التقارير مع الملفات:

1. اذهب إلى **"إدارة المستخدمين"**
2. اضغط على تاب **"التقارير اليومية"**
3. ستشوف كل التقارير مع:
   - اسم الموظف
   - عنوان التقرير
   - محتوى التقرير
   - **الملفات المرفقة** (روابط للتحميل)

#### 2. تحميل الملفات:

- اضغط على اسم الملف لتحميله
- الملفات تفتح في تاب جديد

#### 3. إضافة مهام للموظفين:

1. اذهب إلى **"إدارة المهام"**
2. اضغط **"إضافة مهمة جديدة"**
3. املأ البيانات:
   - عنوان المهمة
   - الوصف
   - تعيين إلى (اختر موظف)
   - الأولوية
   - تاريخ الاستحقاق
4. اضغط **"إضافة"**

## البديل: حفظ الملفات محلياً

إذا لم يكن Supabase Storage متاحاً، يمكنك تعديل `fileUploadService.js`:

```javascript
// استخدام disk storage بدلاً من Supabase
const storage = multer.diskStorage({
  destination: (req, file, cb) => {
    cb(null, "uploads/daily-reports/");
  },
  filename: (req, file, cb) => {
    const uniqueName = `${Date.now()}-${file.originalname}`;
    cb(null, uniqueName);
  },
});

const upload = multer({ storage });
```
