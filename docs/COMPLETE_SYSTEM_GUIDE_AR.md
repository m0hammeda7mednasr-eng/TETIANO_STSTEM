# 🎯 دليل النظام الكامل - Shopify Store Management

## 📋 نظرة عامة

نظام إدارة متجر Shopify متكامل مع:

- ✅ نظام صلاحيات متقدم
- ✅ إدارة مستخدمين احترافية
- ✅ تقارير يومية للموظفين
- ✅ طلبات صلاحيات
- ✅ حساب الأرباح والتكاليف
- ✅ Dashboard احترافي

---

## 🗂️ هيكل النظام

### للـ Admin (المدير):

1. **لوحة التحكم** - إحصائيات شاملة + الأرباح
2. **الطلبات** - إدارة كاملة
3. **المنتجات** - مع سعر التكلفة والربح
4. **العملاء** - قاعدة بيانات العملاء
5. **التقارير** - تقارير إحصائية للمتجر
6. **المستخدمين** - إدارة شاملة:
   - قائمة المستخدمين وصلاحياتهم
   - طلبات الصلاحيات (موافقة/رفض)
   - التقارير اليومية لكل الموظفين
7. **الإعدادات** - ربط Shopify

### للـ User (المستخدم):

1. **لوحة التحكم** - إحصائيات أساسية (بدون أرباح)
2. **الصفحات المسموحة** - حسب الصلاحيات
3. **تقاريري اليومية** - كتابة تقارير يومية
4. **طلب صلاحية** - طلب صلاحيات إضافية

---

## 🔐 نظام الصلاحيات

### الصلاحيات المتاحة:

```
✓ can_view_dashboard      - عرض لوحة التحكم
✓ can_view_products       - عرض المنتجات
✓ can_edit_products       - تعديل المنتجات
✓ can_view_orders         - عرض الطلبات
✓ can_edit_orders         - تعديل الطلبات
✓ can_view_customers      - عرض العملاء
✓ can_edit_customers      - تعديل العملاء
✓ can_manage_users        - إدارة المستخدمين
✓ can_manage_settings     - إدارة الإعدادات
✓ can_view_profits        - عرض الأرباح
```

### كيف يعمل النظام:

1. الـ **Sidebar يخفي** الصفحات اللي المستخدم مالوش صلاحية عليها
2. الـ **Backend يتحقق** من الصلاحيات قبل أي عملية
3. المستخدم يقدر **يطلب صلاحيات** جديدة
4. الـ Admin **يوافق أو يرفض** الطلبات

---

## 📊 صفحة إدارة المستخدمين (للـ Admin)

### التابات الثلاثة:

#### 1️⃣ المستخدمين

- عرض كل المستخدمين
- إضافة مستخدم جديد
- تعديل صلاحيات أي مستخدم
- حذف المستخدمين (ماعدا الـ Admins)

#### 2️⃣ طلبات الصلاحيات

- عرض كل الطلبات (معلقة، موافق عليها، مرفوضة)
- عداد للطلبات المعلقة
- موافقة/رفض مع إضافة ملاحظات
- تحديث تلقائي للصلاحيات عند الموافقة

#### 3️⃣ التقارير اليومية

- عرض تقارير كل الموظفين
- تفاصيل كاملة لكل تقرير
- المرفقات (صور، Excel، PDF)
- حالة التقرير (مسودة، مُرسل، تمت المراجعة)

---

## 📝 التقارير اليومية

### للمستخدم:

- صفحة "تقاريري اليومية"
- كتابة تقرير يومي عن الشغل
- رفع مرفقات (صور، ملفات)
- تعديل وحذف التقارير

### محتوى التقرير:

```
- التاريخ
- العنوان
- الوصف
- المهام المنجزة
- ملاحظات إضافية
- المرفقات (صور، Excel، PDF، إلخ)
```

---

## 💰 نظام الأرباح

### في المنتجات:

```sql
- price          (السعر للعميل)
- cost_price     (سعر التكلفة)
- profit         (الربح = price - cost_price)
- profit_margin  (نسبة الربح %)
```

### في Dashboard:

- إجمالي الأرباح
- نسبة الربح
- أكثر المنتجات ربحاً
- **يظهر فقط للـ Admin** أو من له صلاحية `can_view_profits`

## 🚀 خطوات التشغيل

### 1. إعداد قاعدة البيانات:

```sql
-- نفذ الملفات بالترتيب:
1. ADD_USER_ROLES_PERMISSIONS.sql
2. ADD_DAILY_REPORTS_AND_ACCESS_REQUESTS.sql
3. ADD_COST_PRICE_TO_PRODUCTS.sql
```

### 2. تحديث المستخدمين:

```sql
-- اجعل كل المستخدمين admins (للتجربة):
UPDATE users SET role = 'admin';

-- أو اجعل مستخدم واحد admin:
UPDATE users SET role = 'admin' WHERE email = 'your@email.com';
```

### 3. تشغيل Backend:

```bash
cd backend
npm start
```

### 4. تشغيل Frontend:

```bash
cd frontend
npm start
```
