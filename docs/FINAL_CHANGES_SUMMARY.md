## 🔄 سير العمل (Workflow)

### سيناريو 1: إضافة موظف جديد

```
1. المدير → /users
2. زر "إضافة مستخدم جديد"
3. ملء البيانات + اختيار الصلاحيات
4. حفظ
5. الموظف يستلم بيانات الدخول
6. الموظف يسجل دخول ويرى فقط الصفحات المسموح بها
```

### سيناريو 2: طلب صلاحية جديدة

```
1. الموظف → /request-access
2. اختيار الصلاحية + كتابة السبب
3. إرسال الطلب
4. المدير يرى إشعار في Dashboard
5. المدير → /users → تبويب "طلبات الصلاحيات"
6. موافقة أو رفض + إضافة ملاحظات
7. الموظف يرى النتيجة
8. إذا تمت الموافقة، الصفحة تظهر تلقائياً في Sidebar
```

### سيناريو 3: كتابة تقرير يومي

```
1. الموظف → /my-reports
2. زر "إضافة تقرير جديد"
3. ملء التفاصيل + رفع ملفات
4. حفظ كمسودة أو إرسال
5. المدير يرى التقرير في Dashboard
6. المدير → /users → تبويب "التقارير اليومية"
7. عرض التفاصيل الكاملة + تحميل المرفقات
```
