# دليل المدير الشامل - نظام إدارة متجر Shopify

## 🎯 نظرة عامة

هذا النظام الاحترافي يوفر إدارة كاملة لمتجر Shopify مع نظام صلاحيات متقدم وتقارير يومية.

---

## 👤 حساب المدير (Admin)

### كيف أعرف حساب المدير؟

قم بتشغيل هذا الأمر في قاعدة البيانات Supabase:

```sql
SELECT id, email, name, role FROM users WHERE role = 'admin';
```

أو يمكنك تسجيل الدخول بأي حساب وإذا كان دوره "admin" ستظهر لك جميع الصفحات الإدارية.

### إنشاء حساب مدير جديد

إذا لم يكن لديك حساب مدير، قم بتسجيل حساب جديد ثم نفذ هذا الأمر في Supabase:

```sql
UPDATE users
SET role = 'admin'
WHERE email = 'your-email@example.com';
```

## 🎨 واجهة المدير (Admin Dashboard)

### ما يراه المدير في لوحة التحكم:

#### 1. **الإحصائيات الرئيسية**

- إجمالي المبيعات
- إجمالي الطلبات
- إجمالي المنتجات
- إجمالي العملاء

#### 2. **طلبات الصلاحيات المعلقة**

- زر للانتقال لصفحة إدارة المستخدمين

#### 3. **التقارير اليومية الأخيرة**

- آخر 5 تقارير من الموظفين
- معاينة سريعة للتقارير
- زر للانتقال لصفحة التقارير الكاملة

#### 4. **الرسوم البيانية**

- اتجاه المبيعات
- اتجاه الطلبات

## 👥 صفحة إدارة المستخدمين (/users)

هذه الصفحة الأساسية للمدير وتحتوي على 3 تبويبات:

### التبويب 1: المستخدمين

**الوظائف:**

- عرض جميع المستخدمين في جدول
- إضافة مستخدم جديد (زر أخضر في الأعلى)
- تعديل صلاحيات أي مستخدم (أيقونة Shield)
- حذف مستخدم (أيقونة سلة المهملات)

**الصلاحيات المتاحة:**

- `can_view_dashboard` - عرض لوحة التحكم
- `can_view_products` - عرض المنتجات
- `can_edit_products` - تعديل المنتجات
- `can_view_orders` - عرض الطلبات
- `can_edit_orders` - تعديل الطلبات
- `can_view_customers` - عرض العملاء
- `can_edit_customers` - تعديل العملاء
- `can_manage_users` - إدارة المستخدمين
- `can_manage_settings` - إدارة الإعدادات
- `can_view_profits` - عرض الأرباح

### التبويب 2: طلبات الصلاحيات

**الوظائف:**

- عرض جميع طلبات الصلاحيات (معلقة، موافق عليها، مرفوضة)
- الموافقة على الطلب مع إضافة ملاحظات
- رفض الطلب مع إضافة ملاحظات
- عند الموافقة، يتم تحديث صلاحيات المستخدم تلقائياً

**حالات الطلبات:**

- 🟡 **معلق (pending)** - في انتظار مراجعة المدير
- 🟢 **موافق عليه (approved)** - تم منح الصلاحية
- 🔴 **مرفوض (rejected)** - تم رفض الطلب

### التبويب 3: التقارير اليومية

**الوظائف:**

- عرض جميع التقارير اليومية من الموظفين
- عرض تفاصيل كل تقرير (عنوان، وصف، مهام منجزة، ملاحظات)
- تحميل المرفقات (صور، Excel، PDF، Word)
- المدير يشاهد التقارير فقط ولا يكتبها

**حالات التقارير:**

- 📝 **مسودة (draft)** - لم يتم إرساله بعد
- ✅ **مُرسل (submitted)** - تم إرسال التقرير
- 👁️ **تمت المراجعة (reviewed)** - تمت مراجعته من المدير

## 📝 ما يراه المستخدم العادي (User)

### في Sidebar (القائمة الجانبية):

المستخدم يرى **فقط** الصفحات التي لديه صلاحية عليها:

- ✅ لوحة التحكم (إذا كان لديه `can_view_dashboard`)
- ✅ الطلبات (إذا كان لديه `can_view_orders`)
- ✅ المنتجات (إذا كان لديه `can_view_products`)
- ✅ العملاء (إذا كان لديه `can_view_customers`)
- ✅ **تقاريري اليومية** (متاح للجميع)
- ✅ **طلب صلاحية** (زر أصفر في أسفل القائمة)

### صفحة تقاريري اليومية (/my-reports)

**الوظائف:**

- كتابة تقرير يومي جديد
- إضافة عنوان ووصف
- كتابة المهام المنجزة
- إضافة ملاحظات
- رفع ملفات (صور، Excel، PDF، Word)
- حفظ كمسودة أو إرسال للمدير
- تعديل أو حذف التقارير

### صفحة طلب صلاحية (/request-access)

**الوظائف:**

- اختيار الصلاحية المطلوبة من قائمة
- كتابة سبب الطلب
- إرسال الطلب للمدير
- متابعة حالة الطلب (معلق، موافق عليه، مرفوض)
- قراءة ملاحظات المدير

## 🔐 كيف يعمل نظام الصلاحيات؟

### 1. المدير (Admin)

- يرى **جميع** الصفحات ما عدا "تقاريري اليومية"
- لا يكتب تقارير يومية (فقط يشاهد تقارير الموظفين)
- يدير المستخدمين والصلاحيات
- يوافق أو يرفض طلبات الصلاحيات

### 2. المستخدم العادي (User)

- يرى فقط الصفحات التي لديه صلاحية عليها
- يكتب تقارير يومية
- يطلب صلاحيات إضافية من المدير
- لا يستطيع إدارة مستخدمين آخرين

### 3. آلية إخفاء الصفحات

الـ Sidebar يتحقق من:

1. دور المستخدم (admin أو user)
2. الصلاحيات الممنوحة له
3. يعرض فقط الصفحات المسموح بها

## 🛠️ سيناريوهات الاستخدام

### سيناريو 1: إضافة موظف جديد

1. المدير يدخل على `/users`
2. يضغط "إضافة مستخدم جديد"
3. يملأ البيانات (الاسم، البريد، كلمة المرور)
4. يختار الصلاحيات المناسبة
5. يحفظ

### سيناريو 2: موظف يطلب صلاحية جديدة

1. الموظف يدخل على `/request-access`
2. يختار الصلاحية المطلوبة
3. يكتب السبب
4. يرسل الطلب
5. المدير يستلم إشعار في Dashboard
6. المدير يدخل على `/users` → تبويب "طلبات الصلاحيات"
7. يوافق أو يرفض مع إضافة ملاحظات
8. الموظف يرى النتيجة في صفحة طلباته

### سيناريو 3: موظف يكتب تقرير يومي

1. الموظف يدخل على `/my-reports`
2. يضغط "إضافة تقرير جديد"
3. يملأ التفاصيل (عنوان، وصف، مهام، ملاحظات)
4. يرفع ملفات إذا لزم الأمر
5. يحفظ كمسودة أو يرسل
6. المدير يرى التقرير في Dashboard
7. المدير يدخل على `/users` → تبويب "التقارير اليومية"
8. يشاهد التفاصيل الكاملة

## 🔍 استكشاف الأخطاء

### المشكلة: لا أرى صفحة معينة في Sidebar

**الحل:**

1. تحقق من دور المستخدم (admin أو user)
2. تحقق من الصلاحيات في قاعدة البيانات:

```sql
SELECT * FROM permissions WHERE user_id = 'USER_ID_HERE';
```

3. امسح الـ cache بالضغط على `Ctrl + Shift + R`

### المشكلة: المدير يرى "تقاريري اليومية"

**الحل:**

- هذا خطأ! المدير لا يجب أن يرى هذه الصفحة
- تحقق من أن `hideForAdmin: true` موجود في Sidebar.jsx

### المشكلة: لا تظهر الطلبات المعلقة في Dashboard

**الحل:**

1. تحقق من أن المستخدم admin
2. تحقق من وجود طلبات معلقة في قاعدة البيانات:

```sql
SELECT * FROM access_requests WHERE status = 'pending';
```
